# Glossar

Dieses Glossar erklärt zentrale fachliche und technische Begriffe der
PalHabit-Dokumentation. Es dient als gemeinsame Begriffsbasis für Team,
Prüfer und weitere Stakeholder.

| Begriff            | Bedeutung                                                                                                                                                                                                                            |
|--------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| PalHabit          | Name der gamifizierten Self-Care-App.                                                                                                                                                                                                |
| User               | Angemeldete Person, die Tagesquests erledigt, Wasser trackt und einen Pal-Fortschritt besitzt.                                                                                                                                   |
| Pal            | Virtuelles Begleittier des Users. Es entwickelt sich abhängig von Fortschritt, Motivation und erledigten Aufgaben.                                                                                                                   |
| Ei                 | Anfangszustand des Pal-Fortschritts. Aus dem Ei entsteht später das sichtbare Partner-Pal.                                                                                                                                   |
| Starter-Pal    | Pal-Partner, der beim Start beziehungsweise nach der Registrierung zugeordnet wird.                                                                                                                                              |
| Current Pal    | Aktuell sichtbares Pal des Users. Es repräsentiert den momentanen Entwicklungsstand.                                                                                                                                             |
| Evolution          | Entwicklung eines Pal in eine nächste Stufe. Sie wird durch Fortschritt beziehungsweise Wachstum ausgelöst.                                                                                                                      |
| Quest              | Tagesaufgabe, die erledigt werden kann und Fortschritt erzeugt.                                                                                                                                                                      |
| Tagesquest         | Aufgabe, die innerhalb eines Tages erledigt werden kann und nach einem Reset wieder offen ist.                                                                                                                                       |
| Quest-Punkte       | Punkte aus erledigten Quests, die in den Pal-Fortschritt einfließen. Technisches Feld: `pendingFeedPoints`.                                                                                                                      |
| Motivation         | Pflegewert des Pal-Partners. Technisches Feld: `happiness`. Sinkt bei Inaktivität oder über den manuellen Testbutton. Bei 0 Motivation wird weiterer Verlust auf Wachstum/XP angerechnet; Quests werden dadurch nicht blockiert. |
| Wachstum           | Fortschritt Richtung nächstes Pal-Level. Technisches Feld: `growth`.                                                                                                                                                             |
| Wasserstand        | Getrunkene Wassermenge, gemessen in Millilitern.                                                                                                                                                                                     |
| Daily Reset        | Tageswechsel-Logik, durch die tägliche Aufgaben wieder geöffnet werden.                                                                                                                                                              |
| Protected Endpoint | API-Endpunkt, der nur mit gültiger Session erreichbar ist.                                                                                                                                                                           |
| Public Endpoint    | API-Endpunkt, der ohne bestehende Session erreichbar ist, zum Beispiel Registrierung oder Login.                                                                                                                                     |
| Session            | Serverseitiger Anmeldezustand eines Users nach erfolgreichem Login.                                                                                                                                                                  |
| Session-Cookie     | Cookie, das den Browser mit der serverseitigen Session verbindet.                                                                                                                                                                    |
| External Service   | Externer Dienst außerhalb der eigenen Anwendung, zum Beispiel Open-Meteo oder PalAPI.                                                                                                                                               |
| PalAPI            | Externe Quelle für Pal-Stammdaten. Im normalen App-Betrieb nicht kritisch, da benötigte Pal-Daten bereits in der Datenbank liegen.                                                                                           |
| Open-Meteo         | Externer Wetterdienst, der Wetterdaten für die Dashboard-Darstellung liefert.                                                                                                                                                        |
| Weather Background | Wetterabhängiger Hintergrund im Dashboard. Wenn Wetterdaten nicht verfügbar sind, wird ein Fallback-Hintergrund verwendet.                                                                                                           |
| Fallback           | Ersatzverhalten, das genutzt wird, wenn ein externer Dienst oder eine Datenquelle nicht verfügbar ist.                                                                                                                               |
| Quality Gate       | Pflichtchecks, die für den Abgabestand erfolgreich sein sollen.                                                                                                                                                                      |
| Quality Hub        | Docker-Dashboard für Quality-Reports, Logs und HTML-Reports.                                                                                                                                                                         |
| Quality Runner     | Einmaliger Docker-Container, der Backend-, Frontend-, Security-, Coverage- und E2E-Checks ausführt.                                                                                                                                  |
| E2E-Test           | End-to-End-Test, der einen sichtbaren Nutzerfluss im Browser prüft.                                                                                                                                                                  |
| Coverage           | Testabdeckung des Codes, gemessen für Backend und Frontend.                                                                                                                                                                          |
| ADR                | Architecture Decision Record; dokumentierte Architekturentscheidung mit Kontext, Entscheidung und Konsequenzen.                                                                                                                      |
